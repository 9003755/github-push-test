## 目标
- 使用你提供的 GitHub API 令牌，连接 GitHub 账号
- 选择或创建目标仓库，并将本地项目 `d:\Ai编程学习\github推送测试` 绑定为远程
- 完成首次推送并验证远程链接与分支追踪

## 前提信息
- 令牌权限：建议最小权限 `repo`（必要时 `workflow` 可选），不包含组织管理
- 远程协议：`https`（令牌认证）或 `ssh`（如你已配置 SSH Key）
- 默认分支：若未指定，使用 `main`

## 执行步骤
- 接入令牌
  - 在当前会话安全注入环境变量 `GITHUB_TOKEN`（不写入任何文件）
  - 验证令牌有效性：调用 GitHub API 获取用户信息与仓库列表
- 仓库选择/创建
  - 若你已有目标仓库：通过 API 拉取仓库元信息，确认 `origin` URL
  - 若需要新仓库：通过 API 在你的账号下创建空仓库（私有或公开）
- 本地仓库检查
  - 检查是否已初始化 Git；如未初始化：`git init`，添加必要文件并提交首个 commit（如已有提交则跳过）
- 绑定远程
  - 设置远程：`git remote add origin <url>`（已存在则 `git remote set-url origin <url>`）
  - 设置上游分支：`git branch -M main`
- 首次推送
  - 执行：`git push -u origin main`
- 校验
  - 查看远程：`git remote -v`
  - 通过 API 获取仓库分支与最新 commit，确认推送成功

## 安全与约束
- 令牌仅用于当前会话调用，不写入代码库或日志
- 不在仓库中提交任何令牌或敏感信息
- 全流程遵循最小权限与可撤销原则

## 需要你提供
- GitHub API 令牌（有效且具备 `repo` 权限）
- 目标仓库：已存在的仓库 URL，或新仓库的名称与可见性（私有/公开）
- 远程协议偏好：`https` 或 `ssh`
- 默认分支是否为 `main`，如需更改请说明
